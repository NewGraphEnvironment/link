# Issue #175 — study-area mapping_code parity (tunnel-free + post-consolidate recompute)

## Outcome

Promoted `with_mapping_code` to a stand-alone tunnel-free `lnk_compare_mapping_code()` export and shipped the **complete provincial methodology** for distributed study-area runs: distribute (any bucketing) → consolidate → post-consolidate recompute → compare. The key methodology finding — captured the hard way — was that drainage-closed + DS-first per-host bucketing **reduces but does not eliminate** cross-WSG access divergence (downstream barriers can be cross-bucket or arrive late in DS-first order); a **post-consolidate recompute is required for correctness**. Built `lnk_access()` as the cheap access-only recompute primitive (twin of `lnk_mapping_code`; 11.86s/WSG vs ~90s full pipeline, identical parity) that makes recompute-ALL bulletproof at ~1× cost — the efficiency keystone (#205).

Authoritative result across all 3 FWCP study areas (50 WSGs on M1 dispatcher + 2 cyphers): **median 99.66%, mean 99.11% mapping_code parity vs bcfp**, 130/148 rows ≥99%. Two genuine recompute-stable divergences fall to taxonomy: UNRS BT 61.8% (Kenney reservoir / dam-override) and SETN salmon 93.7–94.8%. Along the way: caught `id_segment` per-WSG cartesian (#203, branched read by `watershed_group_code` column presence); added ltree GIST/btree indexes to persist streams + barriers (matches fresh's working-table pattern); planner-direction view-vs-table bug (1000× cost blow-up — AOI streams must be a real table with `ANALYZE`, not a view); orphaned `frs_network_features` backend gotcha (`statement_timeout`/`lock_timeout` + `pg_terminate_backend` not `pkill`); cypher-driver shape-drift class fix (#204, column-name-based shape-tolerant COPY in `schema_consolidate`); species-presence filter for closure WSGs (#157 enforced); per-WSG soft-fail to prevent trap-burn data loss.
